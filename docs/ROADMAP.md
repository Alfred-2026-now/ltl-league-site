# 路线图

## P0：项目基础

- 模块化目录结构。
- 联赛数据与页面结构分离。
- 业务计算函数集中管理。
- Git 工作流和架构文档。

## P1：赛事官网完善

- 首页视觉细节和移动端体验优化。
- 队伍详情页或详情弹层。
- 赛程按轮次、日期、赛制筛选。
- 公告按类型归档。
- 规则版本号和更新时间。

## P2：数据维护体验

- 把 `src/data/league.js` 拆成更细的数据文件。
- 增加数据校验脚本，检查选手数量、P 币、排名、赛程格式。
- 为 `src/services/leagueMetrics.js` 补单元测试。
- 增加本地开发服务器和基础 lint。

## P3：联赛管理系统

- 管理员登录。
- 队伍、选手、赛程、赛果后台管理。
- 比赛结果录入和积分自动结算。
- 基于管理员本机 LCU 采集工具的国服战绩导入，详见 [战绩录入与比赛详情设计草案](MATCH_RESULT_IMPORT.md)。
- P 币流水、奢侈税、租借费、追赶基金自动计算。
- 数据审计和规则版本追踪。

## P4：生产化

- 数据库和 API。
- 权限角色：管理员、队长、选手、访客。
- 自动部署和预览环境。
- 错误监控、备份、恢复流程。
